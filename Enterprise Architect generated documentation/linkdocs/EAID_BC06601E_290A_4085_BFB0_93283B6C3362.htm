ID: ID_09
Учасники: Авторизований клієнт (далі – Користувач), Система
Попередні умови: Користувач знаходиться на сторінці бронювання столика.
Резльтат: Користувач забронював столик та результат прецеденту ID_05
Основний сценарій:
  1. 1.
    Система виводить інформацію про столики та вказує, які є вільними;
  2. 2.
    користувач обирає столик та натискає кнопку “Забронювати”;
  3. 3.
    система виводить форму для бронювання столику;
  4. 4.
    користувач заповнює поля форми та натискає кнопку “Підтвердити”;
  5. 5.
    виконання сценарію прецеденту ID_05;
  6. 6.
    система прикріплює до ID замовлення ID столу;
  7. 7.
    система виводить повідомлення про статус операції.
Виключні ситуації: всі столики зайняті або не вдовольняють потреб клієнта, виключні ситуації прецеденту ID_05.
